Why cymel?¶

The following are the goals of cymel.

  • More object-oriented than MEL!

  • Easier than API!

  • Lighter and more nimble than PyMEL!

Making Maya scripting more Python-like was accomplished with pymel (the need for it is described in Why PyMEL?).

Respecting this point, we will solve the problems of pymel and implement useful functions more easily.

  • More comfortable Python scripting

    • Anyway, lightweight and nimble

    • Fast (even faster once the C++ implementation is complete)

    • Mathematics classes are equipped with useful functions such as rigging.

  • Reliable and well behaved

    • Guarantees that all changes are undoable

    • Do not rewrite Maya settings

    • Do not embed callbacks such as scriptJob or MMessage

    • Designed to not hold networked plugs

    • Recommendation not to maintain plugs that depend on instance numbers

  • I don’t aim to wrap everything

    • does not support components

    • commands do not wrap

    • Do not aim for multi-functionality such as adding functions equivalent to API or commands to many classes.
