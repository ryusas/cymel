cymel.qt.maya.getMayaWidget(name=None, uitype=None, cls=<class 'PySide6.QtWidgets.QWidget'>)¶

Get Qt widget corresponding to Maya UI name.

Parameters:

  • name (str) – Full pathname of the UI. If omitted, the current layout will be used.

  • uitype (str) – Valid when name is specified and cls is not None. If known, specify Maya’s UI type as a string. You can specify ‘control’, ‘layout’, ‘window’, or ‘menuItem’. On Windows, I was able to get the layout as a control, but on Linux I was unable to do so (depending on the situation). If not specified, all types will be tried until one is obtained, but if you know it, it is safer to specify it.

  • cls (type) – Class to specify when attempting to retrieve with wrapInstance. If None is specified, getWidgetByPathName will be used.

Return type:

QWidget

Note

If cls is not set to None, a simpler method using the Maya API and wrapInstance will be tried first. In rare cases, wrapInstance will result in OverflowError, in which case getWidgetByPathName will be used. In that case, the appropriate class will be automatically determined.
