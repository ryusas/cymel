class cymel.core.datatypes.vector.Vector¶

class cymel.core.datatypes.vector.Vector(*args)¶

Bases: object

3D vector class.

There is no need to use coordinates and direction vectors differently like the Maya API’s MPoint and MVector; they can be handled in the same way.

:mayaapi2:Like MPoint, it has w that can represent homogeneous coordinates, but if w is 1, it behaves as a sequence of length 3. In that case, w cannot be referenced by index specification, but only as an attribute.

If w is not 1, it behaves as a sequence of length 4. Also, only in that case can you check with str or repr.

Multiplying by Matrix allows translation like MPoint, but to convert without translation like MVector, you can use the xform3 method without setting w to 0.

In other words, unless you use homogeneous coordinate representation, it is best to always use w as 1.

The following values ​​can be specified in the constructor.

  • Vector

  • x, y, z, w

  • x, y, z

  • x, y

  • Sequence up to 4 values

Note

The reason why w behaves in this special way is to ensure interchangeability with other objects such as APIs.

Unless w is explicitly set to something other than 1, it can also be passed to something that does not accept sequences of length 4, such as the MVector constructor.

Also, if you specify a value other than 1 for w, that information will be properly inherited even when exchanging with MPoint or list.

Is it almost the same value?

Parameters:

  • v (Vector) – Vector to compare.

  • tol (float) – Tolerance.

Return type:

bool

Note

:mayaapi2:Note that the behavior is different from isEquivalent of MPoint. MPoint compares the difference distance in cartesianize, but there are problems such as undesirable results when w=0. In the case of this method, simple comparison is performed for each element, which is simple and inexpensive.
