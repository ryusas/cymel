asAA()¶

Alternative name for asAxisAngle.

asAngle(naxis=None, bound=False)¶

Get the rotation angle of the quaternion.

asAxisAngle can also obtain a small angle that is 0, but in that case it does not correspond to the axis obtained by asAxisAngle (which will probably always be (0,0,1) when treated as 0 degrees).

Also, if you know the axis of rotation in advance, you can obtain more accurate values ​​by specifying it.

Parameters:

  • naxis (Vector) – If you know the rotation axis, specify it. It must be parallel to the vector given by asAxisAngle. A reversed axis can also be specified, in which case the reversed angle will be obtained.

  • bound (bool) – Whether to obtain angles within ±π.

Return type:

float

asBend(reverse=False, aim=ImmutableVector(1.000000, 0.000000, 0.000000))¶

Obtain the bending component when the bone is rotated.

Parameters:

  • reverse (bool) – Whether to reverse the order of bending/twisting. By default, the decomposition is performed by bending and then twisting when viewed from the top of the hierarchy, but if you specify True, the decomposition is performed by twisting and then bending.

  • aim (Vector) – Bone direction vector.

Return type:

Quaternion

asD(order=0, correct=False)¶

Alternative name for asDegrees.

asDegrees(order=0, correct=False)¶

Obtain the Euler angle rotation as a list in degrees.

Parameters:

  • order (int) – The rotating order you want to get.

  • correct (bool) – When the obtained Euler angle rotation value is requaternized, it is corrected so that it matches the original sign. Equivalent to EulerRotation.reverseDirection when inappropriate.

Return type:

list

Note

If you want to obtain the unit in degrees, use asEulerRotation.

Warning

If you try to set rotateAxis to a value obtained by specifying true for correct, it may not be set correctly, so it should not be specified for that purpose.

asRoll(aim=ImmutableVector(1.000000, 0.000000, 0.000000))¶

Obtain the twist component when the bone is rotated.

The idea of ​​decomposition is that the same value is obtained whether you bend and then twist, or twist and then bend, so there is no reverse option like asBend or asRollBendHV.

Parameters:

aim (Vector) – Bone direction vector.

Return type:

Quaternion

asRollBendHV(reverse=False, aim=ImmutableVector(1.000000, 0.000000, 0.000000), upv=ImmutableVector(0.000000, 1.000000, 0.000000))¶

In the case of bone rotation, it is separated into three angles: twisting, horizontal bending, and vertical bending.

  • aim (Vector) – Bone direction vector.

  • upv (Vector) – Bone up vector.

[roll, bendH, bendV]

isEquivalent(q, tol=1e-10)¶

Are they approximately equivalent? Reversal of sign is considered equivalent as representing the same rotational attitude.

If you do not want to consider sign reversal as equivalent, use isSignedEquivalent.

Parameters:

  • q (Quaternion) – The quaternion to compare.

  • tol (float) – Tolerance.

Return type:

bool

Note

MQuaternion’s isEquivalent did not treat sign reversal as equivalent before 2015, but changed so that it is considered equivalent after 2015.

In accordance with the current API specifications, cymel consistently considers all versions to be equivalent.

classmethod slerp(p, q, t, spin=0)¶

Perform spherical linear interpolation on quaternions.

Parameters:

  • p (Quaternion) – Starting quaternion.

  • q (Quaternion) – Terminal quaternion.

  • t (float) – Interpolation factor between 0.0 and 1.0. You can also specify outside the range.

  • spin (int) – Spin value. The default 0 is the shortest direction, -1 is the opposite direction, and +1 and -1 make extra laps.

Return type:

Quaternion

slerp0(t, spin=0)¶

Perform unit quaternion and spherical linear interpolation.

Parameters:

  • t (float) – Interpolation factor between 0.0 and 1.0. You can also specify outside the range.

  • spin (int) – Spin value. The default 0 means forward direction, -1 means backward direction, and +1 or -1 means extra laps.

Return type:

Quaternion
