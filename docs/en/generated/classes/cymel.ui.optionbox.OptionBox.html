class cymel.ui.optionbox.OptionBox¶

class cymel.ui.optionbox.OptionBox(*args, **kwargs)¶

Bases: object

Abstract base class for creating option boxes.

When developing an option box, inherit this class, define class attributes such as tool name, option prefix, and default value, and then implement UI construction, value loading/saving, and execution code generation.

Option values ​​are stored in OptionVar, and the values ​​can be read and written from the optionvar attribute created on the instance.

Example of minimal configuration:

import cymel.ui as cmu

class MyOptionBox(cmu.OptionBox):
    TOOL_NAME = 'Test'
    TOOL_VERSION = '1.0.0'
    BUTTON_LABEL = 'Test'
    OPT_PREFIX = 'Test.'
    DEFAULTS = dict(foo=False, bar=1.0)

    def assemblePyCode(self):
        return 'print(dict(%s))' % self.assemblePyCodeWithArgs()

    def createContents(self, tabId):
        if tabId == 1:
            cmu.CheckBoxGrp('foo', ncb=1, l1='Foo', cc=self.onChanged)
            cmu.FloatFieldGrp('bar', l='Bar')

    def loadOptions(self, tabId=None):
        if (tabId == 1 or tabId is None) and self.isTabReady(1):
            cmu.Control('foo').setValue(self.get('foo'))
            cmu.Control('bar').setValue(self.get('bar'))

    def saveOptions(self, tabId=None):
        if (tabId == 1 or tabId is None) and self.isTabReady(1):
            self.set('foo', cmu.Control('foo').getValue())
            self.set('bar', cmu.Control('bar').getValue())

    def updateState(self, tabId=None):
        if (tabId == 1 or tabId is None) and self.isTabReady(1):
            cmu.Control('bar').set('enable', cmu.Control('foo').getValue())

# MyOptionBox().perform(0)  # Execute the command using the current settings.
# MyOptionBox().perform(1)  # Open the Options box.
# MyOptionBox().perform(2)  # Retrieve the command string to be executed using the current settings.

assemblePyCodeWithArgs(modName=None, funcName=None, args=None, kwargs=None, argKeys=None, ignoreKeys=None, qualifyKeys=None, asDict=False, skipDefaultValues=True, commandDefaults={})¶

Helper for generating Python code for execution.

Arguments are assembled in the following order.

  1. The value specified by args becomes a positional argument.

  2. The stored value of the key specified by argKeys is added as a positional argument.

  3. The value specified by kwargs becomes the keyword argument.

  4. Automatically collect keyword arguments from internal optionVar.

Automatic collection considers the following conditions:

  • If skipDefaultValues is True, values ​​that are the same as default values ​​will not be collected.

  • Duplicate keys specified directly in kwargs are not collected.

  • Keys specified in argKeys are not collected as keyword arguments.

  • Do not collect keys contained in ignoreKeys.

  • If qualifyKeys is specified, only those keys will be collected.

Parameters:

  • modName (str) – Module name containing the function. When specified with funcName, returns the import module; module.func(...) format.

  • funcName (str) – Function name to call. If omitted, only the argument list part is returned.

  • args (iterable) – The value of a positional argument that you specify directly.

  • kwargs (dict) – The value of the keyword argument specified directly.

  • argKeys (iterable) – The key to retrieve from optionVar as a positional argument.

  • ignoreKeys (container) – Execution keys to exclude from automatic collection.

  • qualifyKeys (container) – Execution keys to be automatically collected.

  • asDict (bool) – If True, return the keyword argument dictionary instead of the code string.

  • skipDefaultValues (bool) – If True, the same value as the default value will not be automatically collected.

  • commandDefaults (dict) – When skipDefaultValues=True, you can specify the comparison value when the default value on the execution command side is different from the storage DEFAULTS.

Returns:

A Python code string or keyword argument dictionary.

createContents(tabId)¶

Generate UI controls within the tab.

Must be implemented in derived classes. When called, the layout of the target tab has been set by setParent, and it is called only once when an ungenerated tab is displayed for the first time.

Parameters:

tabId (int) – Tab index starting from 1. 0 if tabs are not used.

growUpWindowSize(tabId=None, fitW=False, fitH=False, addW=0)¶

Window resizing process called by show.

If the window size is not large enough for the tab, it will be resized.

Often called after UI construction. Setting fitW or fitH to True will also make adjustments in the direction of decreasing.

Parameters:

  • tabId (int) – Tab index starting from 1. If omitted, the current tab.

  • fitW (bool) – Resize even if the width becomes smaller.

  • fitH (bool) – Resize if the height becomes smaller.

  • addW (int) – Number of pixels to add to the calculated width.

isTabReady(tabId)¶

If the contents of the tab have been generated, setParent returns True.

Use loadOptions and saveOptions to avoid accidentally operating delayed tabs that are not displayed yet.

Parameters:

tabId (int) – Tab index starting from 1.

Return type:

bool

Get a method callback that does not strongly refer to this instance.

Passing a bound method directly to a UI callback extends the instance lifetime, so return a function that calls the target method via a weak reference.

  • args – Fixed positional arguments passed to the method.

  • kwargs – Fixed keyword arguments passed to the method.

reset()¶

Reset option values ​​and update the UI.

To avoid losing the NO_RESETS value in an unsaved state, call saveOptions first, then process resetOptions, loadOptions, and updateState in that order.

resetOptions()¶

Resets retained option values ​​to default.

Keys specified as NO_RESETS will be excluded.

save()¶

Save UI state as an option value.

setParent the current option box layout, then call updateState and saveOptions. Update the status before saving, considering the case where it is applied while editing a numeric field.

saveOptions(tabId=None)¶

Read and save values ​​from the UI.

Must be implemented in derived classes. If tabId is None, it is assumed that all generated tabs will be targeted.

Parameters:

tabId (int) – Tab index starting from 1. If None, then whole.

set(key, val)¶

Save the value.

After converting it to a storage key with keyToStore, delegate the processing to set_.

Parameters:

  • key (str) – Execution key.

  • val – Value to save.

set_(rawkey, val)¶

Save the value by specifying the save key.

Override in a derived class if you want to customize value storage. By default, it is passed as is to the internal OptionVar.

Parameters:

  • rawkey (str) – The key after being converted with keyToStore.

  • val – Value to save.

show()¶

Show option box.

If the UI already exists, bring that window to the front. If not generated, obtains the Maya standard option box area and sets tabs, buttons, and menu callbacks.

showHelp(*args)¶

Display HELP_URL.

Gives priority to Maya’s showHelp, and if it fails, falls back to the Python standard webbrowser.

title()¶

Returns the window title.

You can change it arbitrarily by overriding it.

str

Update the state of the UI, such as enable/disable.

Implement it in a derived class only when necessary. If tabId is None, it is assumed that all generated tabs will be targeted.

tabId (int) – Tab index starting from 1. If None, then whole.
