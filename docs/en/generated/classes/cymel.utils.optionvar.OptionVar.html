class cymel.utils.optionvar.OptionVar¶

class cymel.utils.optionvar.OptionVar(prefix='', defaults={})¶

Bases: object

optionVar wrapper class.

Behaves like dict.

The following types of values ​​can be handled without special handling.

  • None (stored as string ‘___None___’)

  • bool (stored as string ‘___True___’ or ‘___False___’)

  • int

  • float

  • str

  • int sequence (types cannot be mixed)

  • float sequence (types cannot be mixed)

  • str sequence (types cannot be mixed)

  • empty sequence

Furthermore, by setting a translator with the value set in optionVar with setTranslator, it is possible to handle any type of value.

By specifying an arbitrary prefix, the prefix will be added when saving and reading optionVar, and you can limit the range that is treated as a dictionary.

It is also possible to specify a default value for each value. If a default value is specified, it will have a value even if it is not actually saved, and the expansion of optionVar will be suppressed.

Note

When developing a tool, we recommend specifying a prefix unique to that tool and specifying a set of default values ​​to use it as an options dictionary for that tool.

removeDefault(key)¶

Delete default values.

If the current value is the default value, optionVar is written to retain the current value even after the default value is removed.

If you want to completely delete it, you should use del or pop instead of this method.

Parameters:

key (str) – key.

reset(key)¶

Resets the value of the specified key.

If there is a default value, it will be that value; if there is not, there will be no value.

Parameters:

key (str) – key.

resetAll(ignores=None)¶

Reset all values.

If there is a default value, it will be that value; if there is not, there will be no value.

If the prefix is ​​not set, all Maya values ​​will be deleted, which will result in an error as the operation is not allowed.

Parameters:

ignores (iterable) – A list or set of keys to exclude.
