class cymel.core.datatypes.transformation.Transformation¶

class cymel.core.datatypes.transformation.Transformation(val=None, **kwargs)¶

Bases: object

Transformation information class.

Maya’s matrix data type attribute can store transformation information in addition to the matrix value itself. This class supports this, and if the plug has data, you can directly get this object (depending on the information you have, you will get Matrix, Transformation, or None). You can also set this object directly into the plug.

Furthermore, this class has the functionality to compose transformation elements into a matrix and decompose a matrix into transformation elements. Operations are performed automatically through normal python object attribute access. In addition to attributes such as matrix and quaternion, transformation elements have attributes with exactly the same name and corresponding attributes as the transform node attributes.

Each attribute can be accessed by either long or short name, including: The parentheses following the long name represent the short name and value type, respectively.

  • Basic transformation element attributes

    These allow you to compose a matrix or decompose a matrix into these values.

    • translate (t) (Vector)

    • rotate (r) (EulerRotation)

    • quaternion (q) (Quaternion)

    • shear (sh) (Vector)

    • scale (s) (Vector)

    Rotate and quaternion refer to the same element (rotate), but you can set or get it using either method (form).

  • Transformation Qualification Attribute

    Auxiliary attributes that qualify the behavior of the base transformation element.

    It is not possible to directly decompose the matrix into these values, and it is positioned as an auxiliary attribute for decomposition and composition.

    • rotateOrder (ro) (int)

    • segmentScaleCompensate (ssc) (bool)

    • inverseScale (is) (Vector)

    • jointOrient (jo) (Quaternion)

    • rotateAxis (ra) (Quaternion)

    • rotatePivot (rp) (Vector)

    • rotatePivotTranslate (rpt) (Vector)

    • scalePivot (sp) (Vector)

    • scalePivotTranslate (spt) (Vector)

    rotateOrder is the same as order in rotate (EulerRotation), but you can set any rotateOrder and use it to get the rotate of that order.

    The type of jointOrient and rotateAxis is Quaternion because it follows the specifications of Maya’s transformation matrix attribute.

    InverseScale’s short name is conflicts with python’s is if it is not a string, so you can use is_ instead.

  • matrix attributes

    Combined matrix value.

    • matrix (m) (Matrix)

Each attribute value can be freely referenced and set, but it is an immutable value wrapped in immutable. That is, you cannot use methods that modify itself. For example, to reorder r, calling EulerRotation.reorderIt will result in an error, so either set the value obtained by EulerRotation.reorder or set ro.

Setting a base transformation element attribute or a transformation modifier attribute clears the matrix attribute and then calculates the composition when attempting to read the matrix value.

Setting a matrix attribute clears the base transformation element attributes and then calculates the decomposition when attempting to read the element’s value.

For example, depending on your purpose, you can decompose the matrix as follows.

  • To decompose the matrix according to settings such as pivot and jointOrient, set them all, then set m to obtain the values ​​of t, r(q), sh, and s.

  • To change some of the qualifying attributes without changing the matrix, get m and save it, set the qualifying attributes to change, set m again, and get the values ​​of t, r(q), sh, and s.

Furthermore, it has the following functions:

  • You can specify 0 to 1 of the following as arguments to the constructor.

    • Transformation

    • Matrix

    • Quaternion

    • EulerRotation

  • Attribute values ​​can be specified as long or short names in constructor keyword arguments.

  • When setting an attribute value, you can set the value obtained by get from Plug without changing it to the original type. For example, the type of the rotateAxis attribute is Quaternion, but setting a 3-value list of Radians will convert it to a Quaternion appropriately.

  • Can be multiplied by Transformation or Matrix.

  • When evaluated as a sequence, it takes the form of a setAttr command for a matrix data type attribute.
