cymel.core.typeinfo.getInheritedNodeTypes(nodetype, node=None, asSet=False)¶

Get type information inherited by a node type.

Since it is cached, it is much faster than checking with the nodeType command.

You can get all inherited type names including the specified type name. In the case of tuple, they are arranged in ascending order.

Parameters:

  • nodetype (str) – Node type name.

  • node (str) – A name that identifies the actual node. Although not required, if specified, it will be slightly faster when creating new information.

  • asSet (bool) – Get the result with frozenset.

Return type:

tuple or frozenset

cymel.core.typeinfo.isAbstractType(nodetype, node=None)¶

Get whether the node type is an abstract type.

Since it is cached, it is much faster than checking with the allNodeTypes command.

The return value is an integer, where 0 means not an abstract type, 1 means an abstract type, and 2 means a metaclass (exists for things like plug-in interfaces, but is not actually a real node type).

Parameters:

  • nodetype (str) – Node type name.

  • node (str) – A name that identifies the actual node. Although not required, if specified, it will be slightly faster when creating new information.

Return type:

int
