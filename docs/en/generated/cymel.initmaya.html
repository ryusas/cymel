cymel.initmaya¶

Initializing Maya from standalone python (mayapy).

initialize is called with default settings when a module that depends on Maya in cymel is imported, so there is little need to use this module explicitly, but it can be used if you want to customize the initialization process.

Function Details:

cymel.initmaya.addCymelPluginsPathToAllowedlist()¶

Add the cymel plugin path to the trusted list for Maya 2022 or later UI mode.

You can do this without user confirmation by calling it from userSetup.py.

cymel.initmaya.callUserSetupMel()¶

Call userSetup.mel.

Depending on the Maya specifications, the behavior differs depending on the version.

Until 2020, only the highest priority userSetup.mel on the MEL path will be called, but in later versions all will be called in path order.

cymel.initmaya.initCymelPluginsPath()¶

Set the path for the Maya plugin included with cymel.

Even if it is called repeatedly, it will not be set twice.

cymel.initmaya.initMaya()¶

Returns False if Maya has been initialized; initializes and returns True if Maya has not been initialized.

In any case, cymel constants such as MAYA_VERSION are initialized.

During initialization, all userSetup.py on the path are called, but userSetup.mel is not called.

There is no problem even if you call it repeatedly.

Return type:

bool

cymel.initmaya.initMels(userPrefs=True, startup=True, plugins=False, namedCommand=True)¶

Call (source) various initialization MELs.

If you do not perform any processing that depends on Maya settings, there is no problem even if you do not call it.

Since MEL is not called repeatedly once it is called, there is no problem in calling this function repeatedly.

Parameters:

  • userPrefs (bool) – Whether to call MEL related to user preferences. Set to True when plugins=True.

  • startup (bool) – Whether to call initStartup.mel. Set to True when plugins=True.

  • plugins (bool) – Whether to autoload plugins. Plugins are dynamically resolved using the scene’s requires statement, and if they are needed in batch processing, they should be explicitly loaded, and the processing load is large, so this is set to False by default.

  • namedCommand (bool) – Whether to generate named commands. Since this is a concept for connecting mel code to hotkeys, it should normally be unnecessary if there is no UI, but since the processing load is not large, the default is set to True, following pymel.

cymel.initmaya.initialize(mels=False, plugins=False, userSetup=True)¶

Perform all necessary initialization.

The following will take place:

  • initCymelPluginsPath

  • initMaya (If it has already been initialized, it will be False and nothing will be done)

    Once initialized, further

    • If mels=True then initMels (plugins=plugins)

    • If mels=False and plugins=True then initAutoPlugins

    • If userSetup=True then callUserSetupMel

  • initApiImmutables

If it has been initialized, nothing will be done, so there is no problem in calling it repeatedly.

Parameters:

mels (bool) – Whether to call initMels when initMaya is True. If you do not perform processing that depends on Maya settings, there is no problem even if you do not call it, so it is set to False by default.

:param bool plugins

Whether to autoload plugins.

Plugins are dynamically resolved using the scene’s requires statement, and if they are needed in batch processing, they should be explicitly loaded, and the processing load is large, so this is set to False by default.

:param bool userSetup

Whether to call callUserSetupMel when initMaya is True.
