cymel.pyutils.finalizer¶

Finalizer function called when any object is destroyed.

The class special method __del__ has the following problems.

  • Exceptions in __del__ cannot be caught.

  • It is not guaranteed to be called when the interpreter terminates.

  • If there is a circular reference and __del__ is present, it will not be released. (Because if there is a circular reference, gc cannot guarantee the right time to call __del__)

Note

The problem of not being freed in a circular reference situation when __del__ exists has been resolved in python 3.4 and later ( PEP 442 ).
