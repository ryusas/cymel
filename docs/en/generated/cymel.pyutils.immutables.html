cymel.pyutils.immutables.immutable(type_or_obj, *args, **kwargs)¶

Generates a wrapper that makes the object immutable.

Call immutableType to get a wrapper class with default settings, and a new instance wrapped in it is returned.

Parameters:

  • type_or_obj – The class or instance of the object to wrap. Old style classes are not supported. When an instance is specified, the wrapper’s initialization argument is the instance itself, so it only works if a copy constructor is implemented.

  • args (list) – Initialization argument when a class is specified for type_or_obj.

  • kwargs (dict) – Initialization argument when a class is specified for type_or_obj.

Returns:

wrapper object.

>>> import cymel.main as cm
>>> d = cm.immutable(dict, woo=1, foo=2, boo=3)  # construct a immutable dict.
>>> d = cm.immutable({'woo':1, 'foo':2, 'boo':3})  # same result via copying.
>>> d['foo']
2
>>> # d['foo'] = 9   # CymelImmutableError
>>> isinstance(d, dict)
True
>>> d.__class__ is dict
False
>>> cm.immutable({}).__class__ is d.__class__
True
cymel.pyutils.immutables.immutableType(cls, name=None, module=None)¶

Generate a wrapper class that makes the object immutable.

Wrapper classes are cached and reused along with their class name.

The wrapper class generated for the first time for each type is cached using only the type as a key, and if the class name is omitted, it is reused with any name.

The wrapper class overrides the methods that rewrite the object’s attribute values ​​and raises CymelImmutableError when they are called.

Which method to override is automatically determined by checking the presence or absence of special methods, but classes that cannot be covered by this alone are managed by the OPTIONAL_MUTATOR_DICT dictionary, which must be extended as necessary.

Parameters:

  • cls (type) – The original class to be immutable.

  • name (str) – Name of the wrapper class. If omitted, 'Immutable original class name' is used.

  • module (str) – The name of the module that provides the wrapper class. Set in the __module__ attribute. If omitted, the __module__ attribute of the original class is inherited.

Return type:

type

Warning

If both name and module are omitted, the immutable wrapper class first generated for the specified class will be reused, resulting in both being undefined.

Note

OPTIONAL_MUTATOR_DICT needs to list the names of methods that modify instances for each class. This means that you need to extend it for each class you want to support. However, it is not necessary to list all methods. Even if you do not intentionally extend OPTIONAL_MUTATOR_DICT, attributes setting, deletion, assignment operators, etc. are blocked by default. The following cases require enumerating method names.

  • A method that calls an attribute’s method.

  • Methods on types provided by built-in and binary modules.
