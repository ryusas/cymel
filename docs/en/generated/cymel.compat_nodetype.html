cymel.compat_nodetype¶

Support for renaming node types depending on Maya version.

For compatibility between old and new versions, you can get the correct name for that version.

You can get the correct name string by specifying the Maya node type name in the form of an attribute access for the unique instance compat_nodetype_map.

  • Basically, you can get the correct name for that version from the new name after the change.

  • If you specify a node type whose name has been changed and the original name has been replaced with another node, an AttributeError will occur, meaning that the old and new types are not compatible.

  • Although the behavior is inconsistent, if you specify a node type name that is changed and the original name is obsolete, you will get the new correct name.

  • If none of the above apply, you will get what you specified as is.

Classes:

CompatNodeTypeMap

Name converter to ensure compatibility between old and new versions due to node type name changes depending on Maya version.
