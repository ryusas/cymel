cymel.qt.maya.getMayaWidget(name=None, uitype=None, cls=<class 'PySide6.QtWidgets.QWidget'>)¶

Maya UI 名に対応する Qt ウィジェットを取得する。

パラメータ:

  • name (str) -- UI のフルパス名。 省略した場合はカレントレイアウトとなる。

  • uitype (str) --

    name を指定し cls が None でないときに有効。

    分かっていれば、Maya の UI タイプを文字列で指定する。 'control', 'layout', 'window', 'menuItem' のいずれかを指定出来る。

    Windows では layout を control として取得出来たが Linux では出来ないことがあった（ものによる）。 未指定の場合、取得できるまで全タイプが試されるが、 分かっているなら指定した方が確実。

  • cls (type) -- wrapInstance での取得を試みる際に指定するクラス。 None を指定すると getWidgetByPathName が利用される。

戻り値の型:

QWidget

注釈

cls に None を指定しなければ、 初めに Maya API と wrapInstance を用いた簡易的な手法が試される。 まれに wrapInstance は OverflowError となり、その場合、 getWidgetByPathName が使用される。 その場合のクラスは自動的に適切なものに決定される。
